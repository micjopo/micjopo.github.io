Let's talk about how I worked
as an information developer at Big Blue.

About my role

I worked with a team of three information developers (including myself), one technical editor,
and one content manager for the IBM FileNet Services for Lotus Quickr product.

We collaborated in a DITA structured authoring environment, and utilized a component content management system
developed in-house at the company. We used an Eclipse-based information center
to host our documentation on online help systems.

For more information on Eclipse, visit the open-source community home page: https://eclipse.org/org.


For those curious, DITA stands for the Darwin Information Typing Architecture. Quite the name, isn’t it?
You can learn a lot more about DITA by Googling it, visiting this website: http://dita.xml.org,
or reading the DITA Best Practices book, written by DITA advocates from IBM.
Our DITA work environment afforded us robust tools and processes for authoring and publishing content for each product.

Planning, Installing, and Configuring
IBM FileNet Services for Lotus Quickr

The Planning, Installing, and Configuring IBM FileNet Services for Lotus Quickr concept topic
presents high-level overview information about the product, and how it works with IBM Lotus Quickr.

Link to the information center topic:
http://www-01.ibm.com/support/knowledgecenter/SSNW2F_4.5.1/com.ibm.installingp8qkr.doc/qkrpi001.htm

  1. Breadcrumbs of information center navigation to the current topic. Hyperlinks are displayed for parent topics.
  2. Title of the book that contains the current topic, and all nested topics within the book. A DITA map file or a bookmap file is used to nest topics.
  3. Topic title. The title follows DITA authoring guidelines depending on whether the topic is a concept, task, or reference topic. This topic is a concept-type topic.
  4. Short description of the topic, which summarizes the intended user goal covered in the topic content.
    The short description is not a summary of the entire content, which is a common misconception for content developers that are learning how to author in DITA.

The topic title and short description present high-level information about a user scenario.
In this case, they address the prerequisite steps and purpose of using IBM FileNet Service for Lotus Quickr.
The user opens the information center with a question or task in mind, and this specific topic
is for users that need to refer to a starting place for planning, installing, and configuring the product.

  1. The body paragraphs, which explain the features of IBM FileNet Services for Lotus Quickr and describes items to keep in mind when planning, installing, and configuring the product for the first time.
  1. One of the most important features of DITA-based content is the navigation elements used to link topics together—-topics that contain related information pertinent to the user goal.
  2. The topic URL, date updated, and the copyright information displays on every single topic, generated from a template used for the information center.
  1. You can also choose a language for viewing the current topic, and a product version that uses this topic. User comments are vital to continuously improving the user experience of the information center topics. Collecting user comments leads to updating the content for future iterations of the product.
